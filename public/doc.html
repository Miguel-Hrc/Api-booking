Library API Documentation

Overview

This API allows managing users, catways, and reservations. It supports registration, login, CRUD operations on catways, and reservation management.

Base URL: http://localhost:3000

Tutorial

  1. Register: Send a POST /auth/register request with user data.
  2. Login: Use POST /auth/login to obtain authentication.
  3. Create Catway: Send a POST /catways/ with token to create a new catway.
  4. Make Reservation: Use POST /catways/:id/reservations to add a reservation.
  5. Delete Reservation: Use DELETE /catways/:id/reservations/:idReservation.

Examples

Register a user

POST /auth/register
{
  "name": "John",
  "firstname": "Doe",
  "email": "john@doe.com",
  "password": "123456"
}

Create a Catway

POST /catways/
Authorization: Bearer token
{
  "catwayNumber": 1,
  "type": "long",
  "catwayState": "Good"
}

Reserve a Catway

POST /catways/:id/reservations
{
  "clientName": "Jane",
  "boatName": "Sea Queen",
  "checkIn": "2025-05-01",
  "checkOut": "2025-05-05"
}

Glossary

  • Catway: A docking space for boats.
  • Reservation: A booking of a catway for specific dates.
  • Token: JWT used to authenticate protected routes.
  • CRUD: Create, Read, Update, Delete operations on resources.
